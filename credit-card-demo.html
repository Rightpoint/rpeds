Credit Card Custom Component

AEM Forms Custom Component with Subscribe Pattern

Debugging Instructions

  • Open browser DevTools (F12) and go to the Console tab
  • All subscribe events and model changes are logged with [CreditCard], [Subscribe], and [FieldModel] prefixes
  • Use the Debug Controls below to test the subscribe pattern:
    • Add New Card - Triggers an enum change event
    • Clear Selection - Triggers a value change event
    • Log Current State - Outputs the current field model state
  • Select a card to see the value change in the console
